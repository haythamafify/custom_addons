POS ZATCA Async - Odoo 18

Eliminate ZATCA chain_index collisions in POS by moving EDI submission to an async cron.

Odoo 18 Compatible

Problem

High POS traffic causes SerializationFailure due to chain_index locking in the same transaction.

Solution

Generate invoices instantly, then process ZATCA EDI sequentially via background cron.

Result

No lock collisions, reliable chain_index order, and stable POS performance.

Dashboard: ZATCA Sync Monitor

  • Stat Cards for Today: Pending / Sent / Error
  • List view for invoices in to_send or error
  • Manual Force Run button for the cron
  • Menu path: Point of Sale → Reporting → ZATCA Sync Monitor

Technical Notes

  • Based on account.move (Odoo 18) — no account.edi.document.
  • Safe sequential processing to protect ZATCA chain_index integrity.
