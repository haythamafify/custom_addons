✦ Overview
🚫

Hard Block

Raises an error before any negative-balance move is committed to the database.

🏭

Internal Locations

Only checks outgoing moves from internal warehouse locations.

🔀

Per-Category Bypass

Enable Allow Negative Stock on a product category to skip the check.

⚡

High Performance

Single SQL query for all quants; aggregates multi-move batches efficiently.

✦ Key Features
  • 🔒

    Real-Time Validation Guard

    Intercepts stock.move._action_done — the final commit point in Odoo 18 — so no negative move ever reaches the journal entries.

  • 🌍

    Bilingual Arabic Error Messages

    UserError displays in Arabic with product name, available qty, and demanded qty — clear for warehouse staff who work in Arabic.

  • 🏷️

    Category-Level Flexibility

    Add an Allow Negative Stock boolean toggle to any Product Category. Perfect for consignment items or goods managed differently.

  • 📦

    Multi-Move Batch Awareness

    Aggregates the total demanded quantity per (location, product) across all moves in the same picking before comparing with available stock.

  • 🛢️

    Single-Query Performance

    Fetches all relevant stock.quant records in one ORM query regardless of how many moves are being validated simultaneously.

✦ How It Works
1

User Validates a Picking

The warehouse worker clicks Validate on any outgoing stock picking.

2

Moves Are Filtered

The module selects only moves whose source is an internal location and whose product category does not have allow_negative_stock = True.

3

Demand Aggregated & Stock Queried

Total demanded qty per (location, product) is calculated, then compared against a single stock.quant query.

4

Block or Pass

If any product has insufficient stock → UserError in Arabic is raised and nothing is written. Otherwise the standard Odoo flow continues normally.

✦ Installation
  1. Copy the stock_no_negative folder into your Odoo addons path.
  2. Restart the Odoo server.
  3. Activate Developer Mode in Settings.
  4. Go to Apps → Update App List.
  5. Search for "Prevent Negative Stock" and click Install.

⚙️ Optional Configuration: Go to Inventory → Configuration → Product Categories, open any category, tab Logistics, and toggle Allow Negative Stock to exempt that category from the check.
